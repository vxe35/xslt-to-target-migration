
# Modernisation Plan: XSLT → Python + Jinja2 (Google Cloud Run)

## Overview
- **Source UI/UX Technology**: XSLT
- **Target Modern Stack**: Python + Jinja2 (Google Cloud Run)
- **Migration Complexity**: Medium
- **Estimated Timeline**: TBD (depends on codebase size)
- **Category**: Legacy UI/UX → Modern UI/UX

## Recommended Frameworks / Tools
- Python 3.11+
- Jinja2 Templates
- lxml / ElementTree (XML parsing)
- FastAPI on Cloud Run
- Cloud Build CI/CD

## Benefits
- Jinja2 is a direct conceptual replacement — same template inheritance & variable injection
- Python lxml gives full XPath 1.0/2.0 + XSLT execution for incremental migration
- FastAPI on Cloud Run provides serverless autoscaling with zero infrastructure management
- Gemini API (via Vertex AI) can auto-convert XSLT stylesheets to Jinja2 templates
- Significantly faster developer onboarding vs XSLT toolchain

## Considerations
- XSL-FO PDF generation → replace with WeasyPrint or Cloud Document AI
- XSLT 2.0/3.0 features (grouping, sequences) need careful Python equivalents
- Existing .NET XSLT callers can be proxied through Cloud Endpoints during transition
